# 설치 가이드 (INSTALL)

space-log 스킨을 티스토리에 적용하는 방법입니다.

> 티스토리 UI 메뉴 이름은 버전/시점에 따라 조금씩 다를 수 있습니다. 흐름 위주로 따라와 주세요.

---

## 1. 스킨 편집 화면 열기

1. 티스토리 **관리(Manage)** 로 이동
2. **꾸미기 → 스킨 편집(Skin Edit)** 선택
3. 우측 상단 **html 편집** 클릭 → `HTML` / `CSS` / `파일업로드` 탭이 보입니다.

---

## 2. 파일 적용

스킨은 4종류의 핵심 파일로 구성됩니다.

| 파일 | 적용 위치 |
|---|---|
| `skin.html` | **HTML 탭** 에 전체 내용 붙여넣기 |
| `style.css` | **CSS 탭** 에 전체 내용 붙여넣기 |
| `images/script.js` | **파일업로드 탭** 에서 업로드 |
| `images/space-cursor.cur` | **파일업로드 탭** 에서 업로드 |

### 업로드 시 주의 (경로)

- `skin.html` 은 스크립트를 `./images/script.js` 경로로 불러옵니다.
- `style.css` 는 커서를 `./images/space-cursor.cur` 경로로 불러옵니다.
- 티스토리 파일 업로드는 보통 `images/` 폴더에 저장되므로 위 경로와 맞습니다.
- 만약 업로드 후 별 배경이 안 보이거나 커서가 기본값이라면, 실제 업로드된 경로가 `images/` 인지 확인하세요. → [TROUBLESHOOTING.md](./TROUBLESHOOTING.md)

---

## 3. 미리보기 이미지 추가 (직접 작업 필요)

이 저장소에는 미리보기 이미지가 **포함되어 있지 않습니다.**

1. 적용된 스킨의 홈 화면을 캡처하여 미리보기 이미지를 만듭니다.
2. 파일 이름을 `preview.jpg` 로 저장합니다.
3. **파일업로드 탭** 에서 `images/preview.jpg` 로 업로드합니다.
4. `index.xml` 의 `<preview>` 가 이미 `./images/preview.jpg` 를 참조하도록 되어 있습니다.

> 파일이 없어도 블로그는 정상 동작합니다. 스킨 목록/편집기 썸네일만 비어 보입니다.
> 권장 비율은 가로형(예: 1600×1200 내외)이지만, 정확한 사이즈 요구사항은 티스토리 등록 화면에서 한 번 확인하는 것을 권장합니다. *(manual Tistory preview check)*

---

## 4. 사이드바 모듈 설정 (중요)

사이드바 본문은 티스토리 치환자 `[##_sidebar_element_##]` 로 채워집니다.
즉, **사이드바에 보이는 메뉴(카테고리·검색·최근 글 등)는 티스토리 관리자에서 직접 추가해야** 표시됩니다.

1. **꾸미기 → 사이드바** (또는 스킨 설정 내 사이드바 영역) 으로 이동
2. 원하는 모듈을 추가: `카테고리`, `검색`, `최근 글`, `인기 글`, `방문자 수` 등
3. 저장 후 블로그에서 사이드바에 모듈이 나오는지 확인

> 모듈을 하나도 추가하지 않으면 사이드바에 마스코트 영역만 보이고 아래는 비어 있을 수 있습니다.
> `[##_sidebar_element_##]` 가 동작하지 않는 경우의 대안은 [TROUBLESHOOTING.md](./TROUBLESHOOTING.md) 참고.

---

## 5. 저장 및 적용

1. **적용(저장)** 버튼으로 스킨을 저장합니다.
2. 새 글이 없다면 공개 테스트 글을 하나 작성해 둡니다.
3. 홈 / 글 상세 / 카테고리 페이지를 직접 열어 렌더링을 확인합니다.

---

## 6. 적용 후 1차 점검 체크리스트

- [ ] 홈에서 미션 로그 카드(글 목록)가 보인다
- [ ] 글 상세 페이지에서 제목·본문·댓글이 보인다
- [ ] 사이드바에 설정한 모듈이 나온다
- [ ] 모바일에서 ☰ 버튼으로 사이드바가 열리고 닫힌다 (오버레이/ESC 포함)
- [ ] 별 배경과 커스텀 커서가 동작한다
- [ ] 마스코트 성별/피부톤 선택이 저장되고 새로고침 후 유지된다

문제가 있으면 [TROUBLESHOOTING.md](./TROUBLESHOOTING.md) 를 확인하세요.
